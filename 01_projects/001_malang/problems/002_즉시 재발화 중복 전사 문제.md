# 즉시 재발화 중복 전사 문제

## 문제
- 음성 인식 직후 사용자가 비슷한 문장을 바로 다시 말하면 같은 질문이 한 번 더 반복되는 현상.

## 원인
- 전사 가드의 중복 판정이 `speech_started` 누락 경로 중심으로 동작.
- `speech_started` 정상 수신 경로에서는 동일 전사 재수신을 충분히 차단하지 못함.

## 영향
- `user.final` 전이가 중복으로 발생할 수 있음.
- 사용자 체감상 "같은 질문 반복"으로 품질 저하.

## 해결
- `decideTranscriptGuard(...)`에 공통 duplicate 판정 추가.
- `hasSpeechStarted=true` 경로에서도 동일 텍스트 + 짧은 시간 창(`TRANSCRIPT_DEDUPE_WINDOW_MS`)이면 차단.
- 중복 차단 시 `speechStartedAt` 정리.

## 검증
- 인식 직후 같은 문장 연속 발화 시 `USER 무시: 중복 전사(...)` 로그 확인.
- 동일 질문 반복 빈도 감소 확인.
