# 슬롯 값 정책 맵 추가 및 비허용 값 차단

## 배경
- 결제 슬롯에서 `수표` 같은 비허용 값을 말해도 슬롯이 채워진 것으로 처리되어 종료 단계로 넘어가는 흐름이 확인됐다.
- 코드 하드코딩 대신 시나리오별로 운영자가 정책을 바꿀 수 있도록 runtime config 기반 제어가 필요했다.

## 적용 내용
- Prisma 스키마/마이그레이션에 `slot_value_policy_map` 컬럼을 추가했다.
- Admin runtime config DTO/검증/저장 경로에 `slotValuePolicyMap` 필드를 반영했다.
- Scenario runtime 파서에 `slotValuePolicyMap` 정규화 로직을 추가해 세션 변수로 주입했다.
- FSM 슬롯 채움 단계에서 정책을 검사해 비허용 값은 채우지 않고 현재 슬롯 질문을 유지하도록 했다.
- Admin `RoleplayPromptWorkspace` 런타임 편집기에 `slotValuePolicyMap (JSON)` 입력 UI를 추가했다.

## 정책 구조
- 형식: `Record<slot, { allow?: string[]; deny?: string[] }>`
- 판정 규칙:
  - `deny`에 매칭되면 무조건 거부
  - `allow`가 비어 있으면 허용
  - `allow`가 있으면 매칭된 값만 허용

## 예시
```json
{
  "payment": {
    "allow": ["카드", "현금"],
    "deny": ["수표"]
  }
}
```
