# 연속 발화 인식 회귀 조정

## 배경
- 중복 전사 차단 보강 후, 일부 상황에서 연속 발화 인식이 과하게 억제되는 체감 이슈가 발생했다.

## 원인
- `hasSpeechStarted=true` 경로의 중복 판정을 항상 적용하여, 정상적인 연속 발화까지 같은 텍스트로 판단 시 차단될 수 있었다.

## 조정 내용
- `decideTranscriptGuard(...)`의 중복 판정 조건을 조정했다.
  - 변경 전: `hasSpeechStarted=true`이면 `pendingPhase`와 무관하게 중복 차단 가능
  - 변경 후: `pendingPhase !== "none"`일 때만 중복 차단

## 기대 효과
- 사용자가 정상적으로 연속 발화하는 경우 인식 누락을 줄인다.
- 시스템이 처리 중(`analyzing/staff`)인 구간의 중복 재전사만 차단한다.

## 검증
- ESLint 확인: `yarn eslint "app/(dashboard)/roleplay-test-v2/page.tsx"` 통과
