# 즉시 재발화 시 중복 전사 차단 보강

## 배경
- 실제 사용 중 "인식 직후 바로 다시 말했을 때 같은 질문을 반복"하는 현상이 관찰되었다.
- 로그상 `response.done` 이후 짧은 간격으로 동일 전사가 다시 들어오면 FSM이 동일 전이를 다시 수행할 수 있었다.

## 원인
- 기존 전사 가드의 중복 판정은 `speech_started` 누락 경로에서만 동작했다.
- `speech_started`가 정상 수신된 경로에서는 동일 텍스트 재전사를 별도로 걸러내지 못했다.

## 적용 내용
- `decideTranscriptGuard(...)`에 **공통 중복 판정**을 추가했다.
  - 조건: `previousAccepted.text === text` && `now - previousAccepted.at < TRANSCRIPT_DEDUPE_WINDOW_MS`
  - `hasSpeechStarted=true` 경로에서도 동일하게 중복 차단
- 해당 경로에서 중복 판단 시 `speechStartedAt`를 정리하도록 `shouldClearSpeechStarted=true` 처리

## 기대 효과
- 전사 직후 동일 문장이 재수신되어도 `user.final` 중복 전이가 줄어든다.
- 같은 질문 반복 응답 빈도가 감소한다.

## 검증
- ESLint 확인: `yarn eslint "app/(dashboard)/roleplay-test-v2/page.tsx"` 통과
- 수동 확인 포인트:
  - 인식 직후 같은 문장을 연달아 말했을 때 `USER 무시: 중복 전사(...)` 로그 확인
  - 동일 턴에서 질문 반복 횟수 감소 확인
