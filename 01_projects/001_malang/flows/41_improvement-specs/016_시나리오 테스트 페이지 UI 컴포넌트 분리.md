# 시나리오 테스트 페이지 UI 컴포넌트 분리

## 배경
- `roleplay-test-v2/page.tsx`에 UI 섹션이 집중되어 유지보수/가독성이 떨어졌다.
- 상태/이벤트 로직은 페이지에 두고, UI 렌더 블록은 섹션 컴포넌트로 분리해 변경 범위를 줄일 필요가 있었다.

## 적용 내용
- 아래 UI 섹션을 `admin/components/roleplay-test-v2/`로 분리했다.
  - `ControlSection.tsx`
  - `RecentSessionStatsSection.tsx`
  - `HintSection.tsx`
  - `PromptTraceSection.tsx`
  - `ManualEventSection.tsx`
  - `MorphAnalysisSection.tsx`
  - `EventLogSection.tsx`
  - `EndConfirmModal.tsx`
- 공유 UI 타입을 `types.ts`로 분리하고 페이지에서 import해 재사용하도록 정리했다.
- `page.tsx`는 기존 상태/로직은 유지하고, 섹션 렌더만 컴포넌트 호출로 치환했다.
- props drilling 완화를 위해 섹션 전용 Context를 추가했다.
  - `RoleplayTestSectionStateContext`
  - `RoleplayTestSectionActionsContext`
  - 상단 제어 패널 포함 각 섹션 컴포넌트는 context에서 필요한 상태/액션만 읽도록 변경

## 검증
- `npx eslint "app/(dashboard)/roleplay-test-v2/page.tsx" "components/roleplay-test-v2/*.tsx" "components/roleplay-test-v2/types.ts"`
- `yarn build`는 기존 다른 파일(`components/roleplay-realtime/hooks/useRoleplayRealtimeTesterController.ts`) 타입 이슈로 실패(이번 변경 범위 외)
