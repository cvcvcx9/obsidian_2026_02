# 힌트 지연 개선

## 목적
- 콜드 세션에서 첫 유효 힌트/응답까지의 지연을 줄인다.

## 진입 조건
- 유휴 시간이 지난 뒤 세션이 시작된다.

## 메인 플로우
```mermaid
flowchart TD
  A[세션 시작] --> B[웜 상태 확인]
  B -->|웜| C[일반 요청 처리]
  B -->|콜드| D[워밍업 핑 전송]
  D --> E[모델 준비 상태 캐시]
  E --> F[사용자 요청 처리]
  F --> G[힌트/응답 반환]
```

## 예외 분기
- 워밍업 실패 -> 즉시 본 요청을 처리하고 지연 메트릭을 기록한다.
- 캐시 사용 불가 -> 인메모리 웜 플래그로 대체한다.
