# 대화 턴 처리

## 목적
- 사용자 입력부터 최종 응답 반환까지의 엔드투엔드 흐름을 정의한다.

## 진입 조건
- 사용자가 메시지를 전송한다.
- 세션이 존재하고 최근 컨텍스트를 불러올 수 있다.

## 메인 플로우
```mermaid
flowchart TD
  A[사용자 입력] --> B[언어 및 안전성 검사]
  B -->|정상| C[세션 컨텍스트 로드]
  B -->|차단| X[안전한 대체 응답 반환]
  C --> D[프롬프트 구성]
  D --> E[모델 호출]
  E --> F[응답 후처리]
  F --> G[턴 및 메타데이터 저장]
  G --> H[응답 반환]
```

## 예외 분기
- 컨텍스트 로드 실패 -> 최소 컨텍스트로 진행하고 경고 로그를 남긴다.
- 모델 타임아웃 -> 재시도 유도 메시지를 반환한다.
