# 컨텍스트 갱신

## 목적
- 매 턴마다 세션 상태를 갱신/검증해 동일 스텝 반복을 방지한다.

## 진입 조건
- 새로운 사용자 턴이 도착한다.
- 이전 턴 요약과 스텝 인덱스를 조회할 수 있다.

## 메인 플로우
```mermaid
flowchart TD
  A[새 사용자 턴] --> B[최신 세션 상태 로드]
  B --> C[요청 스텝과 완료 스텝 비교]
  C -->|이미 수행| D[확인 질문 반환]
  C -->|신규 스텝| E[활성 계획에 스텝 추가]
  E --> F[갱신 상태 기반 응답 생성]
  F --> G[새 스텝 인덱스 저장]
  G --> H[응답 반환]
```

## 예외 분기
- 상태 버전 불일치 -> 최근 턴 기반으로 재구성하고 복구 이벤트를 기록한다.
- 스텝 인덱스 누락 -> 다음 액션 전 안전한 요약 응답으로 기본 처리한다.
