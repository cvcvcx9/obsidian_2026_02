# 개선 실행 체크리스트

## 원칙
- 중복코드 제거보다 안정성(중복 실행 방지)을 먼저 적용한다.
- 적용 순서는 **Idempotency -> 맵 정리 -> 책임 분리 -> 중복 제거**를 유지한다.

## 1단계. Idempotency 적용 (완료/운영조정만 남음)
- [x] `sendFsmEvent`에 idempotencyKey 옵션 추가
- [x] `sendFsmEvent` 중복 키 TTL 캐시(`processedFsmEventKeys`) 적용
- [x] `applyFsmActions` 중복 실행 방지(`appliedActionKeys`) 적용
- [x] Realtime 이벤트 키 dedupe(`processedRealtimeEventKeys`) 적용
- [x] reconnect/disconnect 시 idempotency 캐시 초기화
- [ ] 서버 레벨 idempotencyKey 연동 방식 확정(API 스펙)
- [ ] idempotency skip 로그 집계 규칙 확정

## 2단계. 맵 정리 일원화 (완료)
- [x] `cleanupByResponseId`, `cleanupByItemId` 함수로 통합
- [x] `done/cancel/delete` 순서 역전 멱등 처리
- [x] stale mapping GC 규칙 적용

## 3단계. 이벤트 책임 분리
- [x] `parseRealtimeEvent` 정규화 함수 분리
- [ ] `routeRealtimeEvent` + `eventRouter` 테이블 전환
- [ ] `response.done` 목적별 핸들러 분리

## 4단계. 중복 코드 제거
- [ ] 전사 가드 로직 공통화
- [ ] action 처리 공통 유틸화
- [ ] 로깅 포맷/키 표준화

## 검증 체크
- [ ] 중복 이벤트 2회 수신 시 side effect 1회만 실행
- [ ] `done -> canceled` / `canceled -> done` 역순에서도 최종 상태 동일
- [ ] 수동 침묵(stage 1/2), 종료 의도 이벤트에서 중복 전이 없음
